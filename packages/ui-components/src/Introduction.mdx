## Overview

The GIROK Design System provides accessible, theme-aware components for the GIROK platform.
All components meet **WCAG 2.1 AAA** standards with 7:1+ contrast ratios.

### Key Features

| Feature | Description |
|---------|-------------|
| **WCAG 2.1 AAA** | All text meets 7:1+ contrast ratio |
| **Dual Themes** | Light ("Clean White Oak") and Dark ("Midnight Gentle Study") |
| **SSOT Tokens** | Single source of truth via Tailwind CSS 4 `@theme` |
| **44px+ Touch Targets** | Meets minimum accessibility requirements |
| **Keyboard Navigation** | Full support with visible focus rings (4px) |
| **React 19** | Built for latest React with ref-as-prop pattern |

## SSOT Utility Classes

All styling uses **SSOT (Single Source of Truth)** tokens. Never use hardcoded values.

### Border Radius

| Utility | Value | Usage |
|---------|-------|-------|
| `rounded-input` | 24px | Inputs, buttons |
| `rounded-widget` | 32px | Widgets, modals |
| `rounded-editorial` | 40px | Cards |
| `rounded-editorial-lg` | 48px | Form cards |
| `rounded-editorial-xl` | 56px | Hero sections |
| `rounded-editorial-2xl` | 64px | Menu cards |

### Typography

| Utility | Usage |
|---------|-------|
| `font-serif-title` | Serif headings (Playfair Display) |
| `font-mono-brand` | Brand labels, monospace text |
| `tracking-brand` | Letter spacing 0.3em |
| `tracking-editorial` | Tight tracking -0.05em for serif |

### Theme Colors

| Utility | Usage |
|---------|-------|
| `bg-theme-bg-page` | Page background |
| `bg-theme-bg-card` | Card/surface background |
| `text-theme-text-primary` | Primary text |
| `text-theme-primary` | Brand accent color |
| `border-theme-border-default` | Interactive borders |

## Accessibility Compliance

### WCAG 2.1 AAA Requirements

| Requirement | Implementation |
|-------------|----------------|
| **Contrast Ratio** | 7:1+ for all text |
| **Touch Targets** | 44x44px minimum |
| **Focus Indicators** | 4px ring with high contrast |
| **Keyboard Navigation** | Full support with skip links |
| **Reduced Motion** | Respects `prefers-reduced-motion` |
| **Screen Readers** | ARIA labels and roles |

## Contributing

### Development Setup

```bash
# Clone repository
git clone git@github.com:beegy-labs/my-girok.git
cd my-girok

# Install dependencies
pnpm install

# Start Storybook
pnpm --filter @my-girok/ui-components storybook
```

### Adding Components

1. Create component in `packages/ui-components/src/components/`
2. Create story file with `.stories.tsx` extension
3. Export from `packages/ui-components/src/index.ts`
4. Update this Introduction if needed
5. Ensure WCAG AAA compliance

### Code Style

- Use SSOT token utilities (never hardcode values)
- Follow CSF 3.0 story format
- Include `autodocs` tag for auto-generated docs
- Add accessibility descriptions in story metadata
