# 국제화 (i18n)

> react-i18next 구현: 한국어를 기본 언어로, 영어를 fallback 언어로 사용합니다.

## 지원 언어

| 코드 | 언어   | 역할          |
| ---- | ------ | ------------- |
| ko   | 한국어 | 기본 언어     |
| en   | 영어   | fallback 언어 |
| ja   | 일본어 | 선택적 지원   |

## 디렉터리 구조

```
apps/web-main/src/i18n/
├── config.ts
└── locales/
    ├── ko.json
    ├── en.json
    └── ja.json
```

## 설정

```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';

i18n.use(initReactI18next).init({
  resources,
  lng: localStorage.getItem('language') || 'ko',
  fallbackLng: 'en',
  interpolation: {
    escapeValue: false, // React already escapes
  },
});
```

## 사용법

### 기본 번역

```typescript
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t, i18n } = useTranslation();

  return <h1>{t('home.title')}</h1>;
}
```

### 언어 변경

```typescript
const handleLanguageChange = (lang: string) => {
  i18n.changeLanguage(lang);
  localStorage.setItem('language', lang);
};
```

## 네임스페이스 조직

| 네임스페이스 | 목적                                        |
| ------------ | ------------------------------------------- |
| common       | 재사용 가능한 문자열 (save, cancel, delete) |
| nav          | 네비게이션 (login, register, menu items)    |
| auth         | 인증 폼 및 메시지                           |
| resume       | 이력서 폼, 리스트, 상세 정보                |
| errors       | 오류 메시지 및 검증                         |

## 보간

```typescript
// Simple interpolation
t('welcome', { name: user.name });
// Translation: "Welcome, {{name}}!"

// Multiple values
t('duration', { years: 2, months: 3 });
// Translation: "{{years}} years {{months}} months"
```

## 최고의 실천법

| 해야 할 일                               | 하지 말아야 할 일           |
| ---------------------------------------- | --------------------------- |
| 모든 언어에 번역 추가                    | 한 언어에만 추가            |
| 계층형 키 사용 (`auth.login.title`)      | 평면 키 사용 (`loginTitle`) |
| `common.*`를 재사용 가능한 문자열에 사용 | 번역 중복                   |
| 모든 지원 언어 테스트                    | 언어 테스트 생략            |

## 새 번역 추가

1. 모든 로케일 파일(ko.json, en.json, ja.json)에 키를 추가합니다.
2. 일관된 키 명명 규칙 사용: `namespace.section.item`
3. 기능별로 번역을 정리합니다.
4. 모든 지원 언어에서 테스트합니다.
