# Internationalization (i18n)

> react-i18next implementation with Korean default and English fallback

## Supported Languages

| Code | Language | Role              |
| ---- | -------- | ----------------- |
| ko   | Korean   | Default language  |
| en   | English  | Fallback language |
| ja   | Japanese | Optional support  |

## Directory Structure

```
apps/web-main/src/i18n/
├── config.ts
└── locales/
    ├── ko.json
    ├── en.json
    └── ja.json
```

## Configuration

```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';

i18n.use(initReactI18next).init({
  resources,
  lng: localStorage.getItem('language') || 'ko',
  fallbackLng: 'en',
  interpolation: {
    escapeValue: false, // React already escapes
  },
});
```

## Usage

### Basic Translation

```typescript
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t, i18n } = useTranslation();

  return <h1>{t('home.title')}</h1>;
}
```

### Changing Language

```typescript
const handleLanguageChange = (lang: string) => {
  i18n.changeLanguage(lang);
  localStorage.setItem('language', lang);
};
```

## Namespace Organization

| Namespace | Purpose                                  |
| --------- | ---------------------------------------- |
| common    | Reusable strings (save, cancel, delete)  |
| nav       | Navigation (login, register, menu items) |
| auth      | Authentication forms and messages        |
| resume    | Resume forms, lists, and details         |
| errors    | Error messages and validation            |

## Interpolation

```typescript
// Simple interpolation
t('welcome', { name: user.name });
// Translation: "Welcome, {{name}}!"

// Multiple values
t('duration', { years: 2, months: 3 });
// Translation: "{{years}} years {{months}} months"
```

## Best Practices

| Do                                         | Don't                        |
| ------------------------------------------ | ---------------------------- |
| Add translations to ALL languages          | Add to just one language     |
| Use hierarchical keys (`auth.login.title`) | Use flat keys (`loginTitle`) |
| Use `common.*` for reusable strings        | Duplicate translations       |
| Test all supported languages               | Skip language testing        |

## Adding New Translations

1. Add the key to all locale files (ko.json, en.json, ja.json)
2. Use consistent key naming: `namespace.section.item`
3. Keep translations organized by feature
4. Test in all supported languages
